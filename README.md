This Python script implements a simple extractive text summarization system using the natural language processing capabilities of the spaCy library. The goal of the task is to take a long piece of input text and automatically generate a shorter version (summary) that retains the most important information. This is a common problem in the field of Natural Language Processing (NLP), where machines are designed to understand and process human language. At a high level, the script follows a frequency-based approach to summarization. Instead of generating new sentences (as done in abstractive summarization), it selects the most relevant sentences directly from the original text. This makes it an extractive summarization technique, which is simpler and computationally efficient. The process begins by loading the English language model en_core_web_sm from spaCy. This model allows the program to tokenize the input text, identify sentence boundaries, and perform linguistic analysis. Once the text is processed into a doc object, the script extracts all tokens (words and punctuation) and begins building a word frequency dictionary. In this dictionary, each word from the text is counted based on how often it appears. However, not all words are considered important. Common words like “is,” “the,” and “and” (known as stop words) are removed using spaCy’s predefined STOP_WORDS. Additionally, punctuation marks are ignored. This filtering step ensures that only meaningful words contribute to the scoring process. After calculating raw frequencies, the script normalizes them by dividing each word’s frequency by the maximum frequency found in the text. This normalization step scales all values between 0 and 1, making the scoring more balanced and preventing very frequent words from dominating excessively. Next, the script splits the text into individual sentences using spaCy’s sentence boundary detection. Each sentence is then scored based on the sum of the normalized frequencies of the words it contains. In other words, a sentence receives a higher score if it includes more important (frequent) words. Once all sentences are scored, the script determines how many sentences to include in the final summary. This is controlled by the parameter per, which represents the percentage of the total sentences to keep. For example, if per=0.2, the summary will contain 20% of the original sentences. Importantly, the code ensures that at least one sentence is always selected, even if the input text is very short. To extract the top sentences, the script uses the nlargest function from Python’s heapq module. This function efficiently retrieves the highest-scoring sentences without needing to sort the entire list. The selected sentences are then converted back into text, stripped of extra whitespace, and joined together to form the final summary.
The example provided in the script demonstrates how the summarizer works on a paragraph about NLP and text summarization itself. The function summarize_text is called with a parameter of per=0.2, meaning only 20% of the sentences are included in the output. The program then prints both the original input text and the generated summary. Overall, this task showcases a fundamental NLP technique for reducing large volumes of text into concise summaries. While the approach is relatively simple and does not capture deep semantic meaning like advanced transformer-based models, it is effective for many practical applications such as document summarization, news aggregation, and content preview generation. It also serves as a good starting point for understanding more complex summarization algorithms.
